O Storage Manager for Proxmox (SMP) oferece provisionamento de storage e gerenciamento de ciclo de vida para máquinas virtuais em ambientes Proxmox que utilizam sistemas de storage NetApp ONTAP. O SMP permite que os administradores gerenciem o storage diretamente, simplificando as operações por meio de provisionamento automatizado, backup, restauração e clonagem.



== Saiba mais sobre o Gerenciador de Storage para Proxmox

O Storage Manager for Proxmox (SMP) oferece completamente gerenciamento de storage para máquinas virtuais em ambientes Proxmox que utilizam NetApp sistemas de storage. Ao permitir que os administradores gerenciem o storage diretamente, o SMP simplifica as tarefas de gerenciamento de storage e de dados em ambientes Proxmox.



=== Principais características

* *Gerenciamento de storage completamente*: O SMP oferece gerenciamento de ciclo de vida completo para máquinas virtuais, desde o provisionamento de storage até backup e recuperação.
* *Gerenciamento de storage*: Os administradores do Proxmox podem gerenciar o storage diretamente, simplificando as operações e reduzindo a complexidade.
* *Instalador independente*: O SMP é fornecido como um instalador independente que se conecta perfeitamente aos nós do Proxmox por meio de APIs.




=== Capacidades

Cada componente do SMP foi projetado para aumentar a eficiência do gerenciamento de storage:

* *Provisionamento de storage*: Aloque e gerencie facilmente recursos de storage para hosts Proxmox e máquinas virtuais associadas.
+
** *Provisionamento de storage NFS*: Configure e configure o storage NFS do ONTAP para ser usado com seu ambiente Proxmox.
** *Provisionamento de storage iSCSI*: implemente LVM espesso com multipath para um gerenciamento de storage iSCSI robusto e eficiente.


* *Backup e recuperação*: Realize backups confiáveis e recuperações rápidas para garantir integridade de dados e disponibilidade.
+
** *Backups consistentes com falhas e consistentes em arquivos*: Garanta a integridade de dados configurando mecanismos de backup consistentes com falhas e consistentes em arquivos.
** *Operações de restauração de VM*: Execute processos de restauração de máquina virtual para recuperar dados e manter a disponibilidade do sistema.


* *Eficiência de storage*: ONTAP oferece um conjunto abrangente de tecnologias de eficiência de storage projetadas para minimizar o storage necessário para os dados. Essas tecnologias não apenas reduzem o consumo de storage, mas também melhoram o desempenho do sistema, proporcionando:
+
** *Provisionamento rápido de VMs*: simplifique o provisionamento de máquinas virtuais para uma implantação mais rápida.
** *Cópias de conjuntos de dados mais rápidas*: Acelere a duplicação de conjuntos de dados para um gerenciamento de dados eficiente.






=== Pré-requisitos

Certifique-se de ter concluído o seguinte antes de prosseguir:

* Cluster Proxmox Virtual Environment (PVE) 9.x ou posterior em funcionamento.
* ONTAP All Flash FAS (AFF) com os protocolos NFS e iSCSI habilitados no nível SVM. Para obter instruções, consulte link:proxmox-ontap-nfs.html["Configurar datastores NFS com ONTAP"] e link:proxmox-ontap-iscsi.html["Configurar o Proxmox VE 9.x para iSCSI com storage ONTAP"].
+
[NOTE]
====
Para a configuração de LUN iSCSI, os seguintes itens devem estar habilitados:

*Instale multipath e iSCSI em todos os hosts Proxmox:*

[source, shell]
----
apt update
apt install open-iscsi multipath-tools -y
systemctl enable --now iscsid multipathd
----
*Configurar iniciador iSCSI e conectar via iSCSI (em hosts Proxmox):*

Criar ou atualizar `/etc/multipath.conf`:

[source, shell]
----
touch /etc/multipath.conf
----
Ou use:

[source, shell]
----
cat << 'EOF' > /etc/multipath.conf
defaults {
   user_friendly_names yes
   find_multipaths yes
}
blacklist {
   devnode "^sd[a]$"
}
EOF
----
====
* Os hosts Proxmox e o armazenamento ONTAP são acessíveis pela rede a partir do host SMP.

== Configurar o Gerenciador de Storage para Proxmox

Quando o armazenamento e a conectividade com os nós de armazenamento e Proxmox estiverem configurados corretamente, configurar o SMP para automatizar o provisionamento e a proteção de máquinas virtuais, aproveitando as funcionalidades de ONTAP Snapshot e FlexClone.

O primeiro passo é descobrir e adicionar o endpoint de armazenamento ONTAP e, em seguida, os nós Proxmox de destino ao SMP.

.Antes de começar
Certifique-se de que os seguintes pré-requisitos sejam atendidos:

* O SMP está instalado
* A instância SMP possui conectividade com os nós Proxmox e sistemas de storage
* ONTAP cluster para fornecer pools de armazenamento NFS ou iSCSI
* A resolução de DNS é implementada quando se utilizam nomes DNS. Caso contrário, utilize endereços IP para descoberta.
* O ambiente tem versões suportadas de servidores Proxmox Virtual Environment

== Provisionar storage

O provisionamento de um volume como um pool de storage, comumente chamado de datastore, cria um contêiner lógico para máquinas virtuais e seus discos virtuais (QCOW2 ou RAW). O SMP permite o provisionamento de um datastore e sua posterior conexão a um único host ou a todos os hosts dentro de um cluster.

==== Editar um volume provisionado

Utilize o SMP para modificar as configurações de volume após o provisionamento.

.Passos
. Na interface gráfica do usuário (GUI) do SMP, localize o volume na guia *Provision*.
. Clique nos três pontos verticais (⋮) ao lado do volume.
. Selecione *Editar* no menu de contexto para abrir as configurações de volume.
+
.Caixa de diálogo Exibir/Editar Volume
[%collapsible]
====
image::proxmox-smp-image18b.png[Diálogo Editar Volume mostrando opções de configuração de volume]

====
. Ajuste as configurações de volume conforme necessário:
+
** *Redimensionar volume*: Ajusta a capacidade de storage do volume
+

NOTE: O redimensionamento só se aplica a volumes NFS.

** *Snapshot Locking*: Ative para impedir que snapshots sejam excluídos ou alterados
+

NOTE: Isso requer que a licença SnapLock seja instalada no cluster e que o relógio de conformidade do cluster seja inicializado.

** *Proteção Antiransomware*: habilite para detectar proativamente e alertar sobre atividades incomuns que possam indicar um ataque de ransomware
** *Política de exportação*: atualize ou modifique a política de exportação


. Clique em *Salvar* para aplicar as alterações.

|===


=== Comportamento NConnect

Ao usar storage NFS com o Proxmox VE, esteja ciente do comportamento específico do cliente NFS (como usado pelo Debian, o sistema operacional base do Proxmox VE). O kernel do Linux pode reutilizar os parâmetros de montagem da primeira vez que uma exportação NFS é montada em um nó. Se a mesma exportação for montada novamente com opções de montagem diferentes, essas novas opções podem não ser aplicadas.

Como prática recomendada, crie e monte exportações separadas e use endereços IP de servidor NFS diferentes, como LIFs de dados distintas. O SMP detecta automaticamente as conexões existentes, juntamente com seus valores nconnect associados, e aplica o valor nconnect apropriado a quaisquer novas conexões.



== Proteger máquinas virtuais

Quando máquinas virtuais são executadas em datastores NFS com suporte do ONTAP, a abordagem de proteção mais eficiente é permitir que o Proxmox estabeleça a consistência do sistema de arquivos, enquanto o ONTAP captura o ponto de backup real na camada de storage. Os Snapshots do ONTAP fornecem cópias rápidas, com uso eficiente de espaço e de ponto no tempo do volume do datastore, podendo ser usados posteriormente para recuperar datastores inteiros, arquivos individuais ou conteúdo de máquinas virtuais.

O SMP fornece um orquestrador simples para proteção de dados consistente com falhas e consistente em máquinas virtuais, além de gerenciamento de clones. Este software simplifica os ciclos de vida de backup, restauração e clonagem por meio da criação de políticas de backup que podem ser aplicadas a várias tarefas. Essas políticas podem definir agendamento, retenção, replicação e outras funcionalidades. Elas permitem a seleção opcional de snapshots consistentes com arquivos; essa abordagem utiliza a capacidade do QEMU de interromper as operações de E/S antes de criar um snapshot do ONTAP.

O SMP permite proteger um datastore inteiro, onde um volume ONTAP NFS contém várias máquinas virtuais, grupos de máquinas virtuais que podem abranger vários datastores ou em nível de máquina virtual individual.

=== Detalhes do fluxo de trabalho de backup

O fluxo de trabalho de backup de alto nível inclui:

. *Defina o escopo da proteção*: selecione se o alvo da proteção é um datastore (volume ONTAP), um grupo de recursos (vários volumes ou máquinas virtuais) ou uma máquina virtual individual.
. *Confirme o método de consistência*: Escolha como a consistência será criada antes do snapshot do ONTAP:
+
** O agente convidado do QEMU realiza o congelamento/descongelamento para a quiescência do sistema de arquivos no sistema convidado e para a integração com o VSS do Windows.


. *Extrair os arquivos de configuração da máquina virtual*.
. *Tire o Snapshot do ONTAP*: Capture um Snapshot do ONTAP do volume do datastore enquanto as máquinas virtuais estiverem em um estado consistente. Isso se torna a cópia de backup retida.
. *Remover snapshots temporários do Proxmox*: Exclua os snapshots do Proxmox após a conclusão do Snapshot do ONTAP para evitar o crescimento de arquivos delta.
. *Aplicar políticas de retenção e replicação (usando políticas)*: aplicar a política de retenção configurada do ONTAP e replicar o conjunto de Snapshots se a replicação de recuperação de desastres estiver habilitada.


Essa abordagem minimiza o espaço de storage e a largura de banda da rede, armazenando apenas as informações alteradas. Use cópias de Snapshot para criar cópias rápidas da máquina virtual ou do datastore sem afetar o desempenho e, em seguida, envie-as para um sistema secundário usando a tecnologia NetApp SnapMirror para proteção de dados externa e de longo prazo.

=== Restaurar detalhes do fluxo de trabalho

O fluxo de trabalho de restauração varia dependendo do tipo de armazenamento subjacente e envolve as seguintes etapas principais:

. *Defina o escopo da restauração*: selecione se o destino da proteção é um datastore (ONTAP volume), um grupo de recursos (vários volumes ou VMs) ou uma VM individual.
. *Execute a operação de restauração*: O processo de restauração da VM varia de acordo com o protocolo de storage:
+
** *Restaurações baseadas em NFS*: Ao restaurar um volume ou datastore em NFS, o SMP inicia uma operação de restauração de snapshot de volume no volume de destino. Para VMs hospedadas em volumes NFS, o SMP valida os parâmetros de restauração, incluindo ID da VM, nome do snapshot, IDs do site e tipo de storage. Se a VM estiver em execução, o SMP a interrompe consultando seu status e emitindo um comando de parada. Em seguida, o sistema restaura os discos da VM a partir do snapshot usando uma operação de restauração em nível de arquivo.
** *Restaurações baseadas em iSCSI*: Para datastores iSCSI ou LUNs, o SMP restaura o LUN a partir do snapshot, verifica o dispositivo restaurado, realiza uma nova varredura iSCSI em todos os nós do Proxmox e garante a acessibilidade do dispositivo. Para VMs residentes em storage iSCSI, o SMP clona os LUNs do snapshot, mapeia os LUNs clonados para o host Proxmox e realiza a descoberta iSCSI para importá-los. O storage restaurado é então adicionado e as configurações das VMs são atualizadas. Os discos das VMs são movidos de volta para seu storage original, com o progresso rastreado por VM. O processo termina com atividades de limpeza, incluindo a remoção do storage temporário e a exclusão dos LUNs clonados.

=== Detalhes do fluxo de trabalho de clonagem

O SMP automatiza a clonagem rápida de VMs ao transferir o processamento pesado para o ONTAP, que realiza clones no lado do storage em vez de cópias de arquivos no lado do host. Internamente, o SMP utiliza a semântica FlexClone do ONTAP (clones baseados em Snapshot, graváveis e com uso eficiente de espaço), permitindo que os clones sejam criados rapidamente e consumam pouca capacidade inicialmente; ele evita leituras/gravações de dados pelo host Proxmox sempre que possível.

Essa abordagem acelera o provisionamento de VMs ao transferir o trabalho de clonagem para ONTAP, fornecendo clones rápidos, leves e com uso eficiente de espaço, adequados para Dev/Test, laboratórios e ciclos de atualização rápidos.

Por padrão, a clonagem é realizada no sistema de arquivos ativo. O fluxo de trabalho de clonagem também permite a seleção de um snapshot, caso em que as clonagens de arquivos são realizadas usando um novo volume criado a partir do snapshot selecionado.

== Resumo

O Storage Manager para Proxmox oferece uma maneira simplificada e unificada de gerenciar operações de proteção, clonagem e recuperação em ambientes Proxmox. Ao integrar-se diretamente com ONTAP, a solução traz a eficiência, confiabilidade e desempenho do storage de classe empresarial para os fluxos de trabalho diários de VM. Os snapshots com uso eficiente de espaço do ONTAP, as capacidades de clonagem rápida e a replicação incorporada fornecem proteção de dados consistente e previsível, enquanto o SMP simplifica esses recursos em uma interface intuitiva e focada em tarefas.
