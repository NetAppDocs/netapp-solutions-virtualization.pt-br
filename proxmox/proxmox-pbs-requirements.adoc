= Requisitos e considerações de implementação para o servidor de backup Proxmox com armazenamento NetApp ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Antes da implementação, revise os requisitos e considerações de implantação do Proxmox Backup Server (PBS) com armazenamento NetApp ONTAP . Isso garante o planejamento adequado para a implantação do servidor de backup, configuração de armazenamento e conectividade de rede.



== Considerações sobre a implantação

Implante o PBS em hardware dedicado ou como uma máquina virtual no Proxmox VE. Ao implementar o PBS como uma máquina virtual, considere a disponibilidade do servidor de backup caso o host Proxmox VE fique inativo. Implante o PBS em hardware dedicado ou em um host ou cluster Proxmox VE separado para garantir a disponibilidade do servidor de backup durante falhas do host primário.

O conteúdo do backup fica armazenado de forma independente no próprio armazenamento de dados, permitindo a migração do armazenamento de dados para outra instância do PBS, se necessário. Isso garante que os dados de backup permaneçam acessíveis mesmo quando a instância original do PBS estiver indisponível.



== Suporte ao backend de armazenamento

O PBS suporta vários sistemas de armazenamento, incluindo armazenamento local, NFS, iSCSI, FC, NVMe-oF e S3 (versão prévia técnica). A integração com o armazenamento NetApp ONTAP permite que as organizações aproveitem recursos avançados de gerenciamento de dados para melhor proteção e recuperação de dados.

Utilize o ONTAP SnapMirror para replicar o armazenamento de dados PBS em outro sistema ONTAP para recuperação de desastres. Isso protege os dados de backup e permite a restauração após falhas no site.



== Gestão de múltiplos clusters

Uma única instância do PBS pode gerenciar vários clusters Proxmox VE, fornecendo backup centralizado para implantações em larga escala. Utilize namespaces para separar logicamente os dados de backup de diferentes clusters ou tenants, garantindo o isolamento e a segurança dos dados.

Configure políticas de retenção de dados para gerenciar automaticamente o ciclo de vida dos dados de backup e garantir a conformidade com os requisitos da organização. Configure políticas de limpeza para marcar automaticamente dados de backup antigos ou desnecessários para exclusão. Os processos de coleta de lixo recuperam espaço de armazenamento removendo dados não utilizados ou obsoletos do repositório de dados.



== Interfaces de gerenciamento

Gerencie o PBS por meio de uma interface baseada na web, ferramentas de linha de comando ou API REST. Acesse a interface web na porta 8007 usando o URL `https://<pbs-ip-address>:8007`.



== Melhores práticas de configuração de rede

Garanta caminhos de rede redundantes entre o PBS e o armazenamento ONTAP para alta disponibilidade e desempenho. Considere a agregação de links (LACP) para aumentar a largura de banda e a redundância. Configure quadros jumbo (MTU 9000) em todos os dispositivos de rede para melhorar o desempenho do tráfego de armazenamento.

Para NFS, crie uma exportação dedicada para o armazenamento de dados PBS com as permissões apropriadas. Para protocolos de bloco, assegure o zoneamento adequado e o mascaramento de LUN para restringir o acesso apenas a hosts PBS autorizados.
